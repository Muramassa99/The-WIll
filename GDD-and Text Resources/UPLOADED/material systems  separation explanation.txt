You did **not** misunderstand the purpose of `BaseMaterialDef`.

You are actually separating two different needs, and both are valid:

## Need 1 — runtime truth

What the game actually stores on a material resource.

## Need 2 — authoring toolbox

A complete “all possible knobs” reference so you can invent materials without forgetting sockets, stat types, or weird edge cases.

Those are **not** the same object.

That is the key.

Your material law already says a Material Atom is the foundation of weight, stat contribution, elemental access, capability unlocks, and visual identity, while the crafted item later becomes a baked profile that gameplay reads. 
And your GDD already says weapons, armor, accessories, and shields all share the same crafting philosophy, with shields being a constrained special case rather than a separate substance system.  

So yes:

# 1) `BaseMaterialDef` should be universal

It should represent **matter itself**, regardless of whether that matter ends up in:

* a weapon
* armor
* a shield
* an accessory
* a trinket
* a ranged-support piece like string/quiver parts

That is correct.

Wood is still Wood.
Iron is still Iron.
Crystal is still Crystal.

The item context changes later. The substance does not.

# 5) What `BaseMaterialDef` should include

You asked for the biggest pile of material-level stats in the game.

That is okay — as long as they are truly **material-level** and not final profile/equipment outputs.

This is the correct first-pass structure:

```gdscript
extends Resource
class_name BaseMaterialDef

@export var base_material_id: StringName = &""
@export var display_name: String = ""

# structural / affinity / keystone
@export var material_family: StringName = &"structural"

# physical truth
@export var density_per_cell: float = 0.0
@export var hardness: float = 0.0
@export var toughness: float = 0.0
@export var elasticity: float = 0.0
@export var brittleness: float = 0.0
@export var edge_retention: float = 0.0
@export var thermal_stability: float = 0.0
@export var corrosion_resistance: float = 0.0
@export var conductivity: float = 0.0

# economy / lifecycle
@export var processing_output_count: int = 6
@export var salvage_priority: int = 0
@export var can_be_processed: bool = true
@export var can_be_salvaged: bool = true
@export var can_be_blueprinted: bool = true

# visual identity
@export var color_group: StringName = &"gray"
@export var metallic_grade: float = 0.0
@export var gloss_grade: float = 0.0
@export var transparency_grade: float = 0.0

# tags / hard hooks
@export var material_tags: Array[StringName] = []
@export var unlock_tags: Array[StringName] = []
@export var block_tags: Array[StringName] = []

# direct numeric stats
@export var base_stat_lines: Array[StatLine] = []

# hidden biases
@export var capability_bias_lines: Array[StatLine] = []
@export var skill_family_bias_lines: Array[StatLine] = []
@export var elemental_affinity_lines: Array[StatLine] = []
@export var equipment_context_bias_lines: Array[StatLine] = []

func has_tag(tag: StringName) -> bool:
	return material_tags.has(tag)

func has_unlock_tag(tag: StringName) -> bool:
	return unlock_tags.has(tag)

func has_block_tag(tag: StringName) -> bool:
	return block_tags.has(tag)
```

That structure is universal enough for weapons, armor, shields, accessories, and trinkets **without** turning material into final gameplay truth.
